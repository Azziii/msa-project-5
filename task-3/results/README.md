# Задание 3. Реализация Distributed Scheduling с Kubernetes CronJob

## Выбранное решение

Для реализации периодической выгрузки данных выбран Kubernetes CronJob.

Причины выбора:

* инфраструктура уже использует Kubernetes;
* задача запускается по расписанию;
* отсутствует необходимость в сложной оркестрации;
* Kubernetes автоматически создаёт и завершает Job;
* решение легко масштабируется и интегрируется с мониторингом.

## Реализованный функционал

Каждый день в 20:00 запускается CronJob.

CronJob:

1. Подключается к PostgreSQL.
2. Выполняет SELECT из таблицы shipments.
3. Экспортирует результат в CSV.
4. Сохраняет файл в каталог контейнера.
5. Завершает выполнение.

## Проверка

Проверка выполняется в Minikube:

```bash
kubectl get cronjobs
kubectl get jobs
kubectl logs <pod>
```
